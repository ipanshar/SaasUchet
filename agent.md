# agent.md

Этот файл задаёт правила работы AI-агента в репозитории `SaasUchet`.

## Назначение

- Файл дополняет `CLAUDE.md` и фиксирует отдельные рабочие правила для агента.
- При расхождении приоритет отдаётся более свежим прямым указаниям пользователя.

## Обязательные правила

1. При разработке агент не пишет тесты и не запускает тесты.
2. Проверку изменений пользователь выполняет вручную.
3. После каждого завершённого сеанса разработки агент должен сделать `git commit` и `git push` в ветку `main`.
4. Сообщение коммита должно быть на русском языке, кратко описывать выполненную работу и быть не длиннее 200 символов.
5. Агент не должен включать в коммит чужие или несвязанные изменения без явного разрешения пользователя.

## Примечание

- Если в рабочем дереве уже есть посторонние изменения, агент коммитит только свои файлы или согласовывает дальнейшие действия с пользователем.
